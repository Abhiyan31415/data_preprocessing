1. Objective

The objective of this laboratory exercise is to design and implement a comprehensive data preprocessing pipeline for a Nepali-English code-switched (Neplish) Automatic Speech Recognition (ASR) system. The specific goals are:

  • Parse raw transcript files containing timestamps and mixed Nepali-English text
  • Segment audio recordings into smaller, manageable clips based on transcript timestamps
  • Clean and normalize transcripts by resolving bracket annotations and removing noise markers
  • Create a structured HuggingFace dataset with appropriate train/validation/test splits
  • Apply data augmentation techniques to increase the diversity and size of the training set
  • Generate comprehensive statistics and visualizations for analysis and presentation

2. Introduction

Code-switching is a linguistic phenomenon where speakers alternate between two or more languages within a single conversation or utterance. In Nepal, code-switching between Nepali and English (often called "Neplish") is extremely common, especially in academic, technical, and urban settings. This presents unique challenges for Automatic Speech Recognition systems, which are typically trained on monolingual data.

Traditional ASR systems struggle with code-switched speech because they lack training data that captures the natural patterns of language mixing. The phonetic, prosodic, and syntactic variations that occur at code-switch points require specialized training data and models.

This project addresses the data preparation aspect of building a Neplish ASR system. We process raw audio recordings of technical presentations (containing natural code-switching) and their corresponding transcripts to create a clean, augmented dataset suitable for fine-tuning models like OpenAI's Whisper.

3. Methodology

3.1 Dataset Description

The raw dataset consists of 3 audio files in AAC format with corresponding transcript files in TXT format. The recordings are technical presentations by 3 speakers: aashraya, mukesh, praja.

The transcript files use different timestamp formats that required handling:

  • [MM:SS MM:SS] > text - Text on the same line after the marker
  • [M:MM - M:MM] - Timestamp on one line, text on the following line
  • Speaker labels like Speaker 1, Speaker 2 for multi-speaker recordings

3.3 Audio Segmentation

Audio files are segmented based on the parsed timestamps and converted to a format compatible with Whisper and other ASR models:

  • Sample Rate: 16 kHz (downsampled from original)
  • Channels: Mono (single channel)
  • Format: WAV (uncompressed)
  • Bit Depth: 16-bit PCM

3.4 Transcript Cleaning

The raw transcripts contain various bracket annotations that must be resolved for ASR training. The following cleaning rules were applied:

Pattern Action Example
[NOISE], [noise] Remove entirely "Hello [NOISE] world" → "Hello world"
[<unk>], [<sil>] Remove entirely "The [<sil>] end" → "The end"
[English/Nepali] Keep English word "[algorithm/एल्गोरिदम]" → "algorithm"
[<rep>/word] Keep the word (repetition) "[<rep>/the]" → "the"
[word/<unk>] Keep the word "[hello/<unk>]" → "hello"
[MM:SS] (inline) Remove timestamps "text [00:15] more" → "text more"

3.5 Data Augmentation

To increase the diversity of the training data and improve model robustness, five augmentation techniques were applied. Each training sample receives exactly one random augmentation:

Technique Parameters Purpose
Gaussian (White) Noise SNR: 20-40 dB Simulate background noise with equal power across frequencies
Pink (1/f) Noise SNR: 20-40 dB Simulate natural ambient noise (more power in lower frequencies)
Tempo Change Rate: 0.93x - 1.07x Simulate different speaking speeds without pitch change
Pitch Shift Range: ±1.5 semitones Simulate voice variations without tempo change
Volume Perturbation Gain: ±4 dB Simulate recording level variations

5. Discussion

The preprocessing pipeline successfully transformed raw audio recordings and transcripts into a structured dataset suitable for ASR model training. Several key observations emerged from this work:

Complete Code-Switching: The finding that 100% of segments contain code-switching is significant. It indicates that the source material (technical presentations) naturally exhibits heavy language mixing, which is representative of how Neplish is spoken in academic and professional settings. This makes the dataset particularly valuable for training models that must handle frequent code-switch points.

Technical Domain: The vocabulary analysis reveals that the content is primarily from technical presentations covering algorithms (Traveling Salesman Problem, simulated annealing), traffic systems, and fuzzy logic. While this provides good coverage of technical Neplish, future work should include more diverse domains (casual conversation, news, etc.) for a more generalizable model.

Minimal Cleaning Required: Only 2.2% of characters were removed during cleaning, indicating that the original transcripts were of high quality with minimal noise annotations. This suggests careful original transcription work.

Augmentation Effectiveness: The 2x augmentation factor provides additional training diversity while maintaining the integrity of the original transcripts. The balanced distribution of augmentation types ensures the model will be exposed to various acoustic variations.

Dataset Size Limitations: With only 49 segments totaling 14.9 minutes, this dataset is relatively small for ASR training. However, it can serve as a valuable fine-tuning dataset for pre-trained models like Whisper, which have already learned general speech patterns from large-scale data.
